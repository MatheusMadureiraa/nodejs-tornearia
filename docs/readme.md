## Implementação
O projeto foi desenvolvido utilizando a metodologia ágil Scrum, adaptada para um contexto de desenvolvimento individual (Scrum solo), mantendo organização por entregas incrementais e controle de tarefas.

### 🧭 Organização em Sprints

O desenvolvimento foi dividido em 6 sprints principais, seguindo uma evolução incremental do sistema:
#### 🟦 Sprint 1 — Planejamento

Definição do escopo, requisitos e estrutura inicial do sistema.

#### 🟨 Sprint 2 — Arquitetura

Definição da arquitetura, organização do backend e estrutura do banco de dados.

#### 🟧 Sprint 3 — Backend

Desenvolvimento da API e regras de negócio com Node.js e Express.

#### 🟩 Sprint 4 — Frontend

Criação da interface do sistema utilizando HTML, CSS e JavaScript.

#### 🟪 Sprint 5 — Integração e Testes

Integração entre frontend, backend e banco de dados, além de testes manuais com Insomnia.

#### 🟥 Sprint 6 — Testes Automatizados e MVP

Implementação de testes automatizados com Playwright e validação final do MVP.


### 📊 Gestão do Projeto
Tarefas organizadas utilizando Kanban
Funcionalidades documentadas como issues no GitHub
Commits vinculados às entregas de cada funcionalidade
Controle incremental de evolução do sistema
